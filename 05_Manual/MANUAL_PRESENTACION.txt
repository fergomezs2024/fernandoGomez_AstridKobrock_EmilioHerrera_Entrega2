PARTE 1: COMO LEVANTAR EL PROYECTO
================================================================================

Necesitas instalar 2 cosas antes de empezar:

  1. Python (el lenguaje de programacion que usa el proyecto)
     Descarga: https://www.python.org/downloads/
     IMPORTANTE: cuando lo instales, marca la casilla "Add Python to PATH"

  2. Git (programa para descargar el codigo del repositorio)
     Descarga: https://git-scm.com/downloads
     Instalar con las opciones por defecto, no hay que cambiar nada.

Una vez instalados, sigue estos pasos:

  PASO 1 - Abrir la terminal:
     En Windows: buscar "PowerShell" o "CMD" en el menu de inicio y abrirlo.
     Te aparece una ventana negra donde puedes escribir comandos.

  PASO 2 - Descargar el proyecto:
     Escribir estos comandos en la terminal (uno por uno, dando Enter):

     git clone <url-del-repositorio>
     cd GestionDeDatos

     "git clone" descarga todo el proyecto a tu computadora.
     "cd" entra a la carpeta del proyecto.

  PASO 3 - Crear un entorno virtual:
     Esto es como crear una "burbuja" para que las librerias del proyecto
     no se mezclen con otras cosas de tu computadora. Es opcional pero
     recomendado.

     python -m venv venv

     Despues activarlo:
       En Windows:    venv\Scripts\activate
       En Mac/Linux:  source venv/bin/activate

     Sabes que funciono si ves (venv) al inicio de la linea en la terminal.

  PASO 4 - Instalar las librerias que usa el proyecto:
     pip install -r 02_Ingesta_y_Pipeline/requirements.txt

     "pip" es el instalador de librerias de Python.
     "requirements.txt" es un archivo que lista todo lo que necesita:
       - pandas (para leer y manipular datos, como un Excel potente)
       - scikit-learn (para los modelos de inteligencia artificial)
       - matplotlib y seaborn (para generar graficos)
       - pytest (para ejecutar pruebas automatizadas)

  PASO 5 - Ejecutar el proyecto:
     python 02_Ingesta_y_Pipeline/analisis.py

     Esto corre TODO el proyecto de principio a fin. Vas a ver en la
     terminal como va pasando por cada etapa. Tarda menos de 5 segundos.

     Cuando termine, se crea la carpeta "03_EDA_Resultados/" con estos archivos:
       - log_ejecucion.log         -> registro detallado de todo lo que paso
       - datos_limpios.csv         -> los datos ya procesados y limpios
       - prestamos.db              -> base de datos con los datos cargados
       - reporte.txt               -> resumen con numeros y resultados
       - distribucion_target.png   -> grafico: cuantos pagaron vs no pagaron
       - distribucion_edad.png     -> grafico: edades de los solicitantes
       - tasa_interes_por_status.png -> grafico: tasa de interes segun estado
       - arbol_decision.png        -> grafico: como el modelo toma decisiones
       - equidad_genero.png        -> grafico: si el modelo es justo por genero

  PASO 6 - Ejecutar las pruebas automatizadas (tests):
     pytest 02_Ingesta_y_Pipeline/test_analisis.py -v

     Esto verifica que los datos estan bien. Deberian salir 5 tests
     en verde (PASSED). Si sale algo en rojo, algo fallo.

  PASO 7 - Ver el dashboard (pagina web visual):
     Ir a la carpeta del proyecto y abrir "04_Pagina_Web/dashboard.html"
     Se abre en el navegador (Chrome, Firefox, Edge, etc.)
     Ahi esta TODO explicado visualmente con graficos.

  SI ALGO FALLA:
     Probar estos comandos:
       python -m pip install --upgrade pip
       python -m pip install -r 02_Ingesta_y_Pipeline/requirements.txt
     Si sigue fallando, verificar que Python esta instalado:
       python --version
     Deberia mostrar algo como "Python 3.12.x"

SLIDE 2 - EL PROBLEMA
------------------------------------------------------------------------
Que poner en la slide:

  "Un banco pierde plata cuando le presta a alguien que despues no paga."

  El problema concreto:
    - El banco recibe miles de solicitudes de prestamos
    - Necesita decidir a quien aprobar y a quien no
    - Hoy eso se hace manual o con reglas basicas, y falla seguido

  Nuestra solucion:
    - Tomamos datos historicos de 45,000 solicitudes de prestamos

    (DataOps = metodologia para gestionar datos de forma automatizada)

    FORMA TRADICIONAL:
      - Alguien descarga datos a mano
      - Los limpia en Excel
      - Hace un grafico
      - Lo manda por mail

      - Si cambian los datos, se reprocesa solo

      - Se monitorea con indicadores (KPIs) que avisan si algo falla
      - Cualquiera puede reproducirlo

SLIDE 3 - EL PIPELINE (FLUJO GENERAL)
------------------------------------------------------------------------
Que poner en la slide:

  Que es un pipeline:
    Es una cadena de pasos que se ejecutan en orden, uno tras otro,
    como una linea de ensamblaje en una fabrica. Los datos entran
    "crudos" por un lado y salen "listos para usar" por el otro.

  Mostrar este flujo (se puede hacer con flechas o cajitas):

    DATOS CRUDOS (CSV)
        |
        v
    [1. INGESTA]      -> Leer los datos
        |
        v
    [2. LIMPIEZA]     -> Corregir problemas
        |
        v
    [3. VALIDACION]   -> Verificar que todo esta bien
        |
        v
    [4. CARGA]        -> Guardar los datos limpios
        |
        v
    DATOS LISTOS (CSV limpio + Base de datos)
        |
        v
    [ANALISIS Y MODELOS DE IA]

  Dato importante: cada etapa depende de la anterior.
  Si la ingesta falla, no se puede limpiar.
  Todo queda registrado en un archivo de log.

SLIDE 5 - ETAPA 2: LIMPIEZA Y TRANSFORMACION
------------------------------------------------------------------------
Que poner en la slide:

  Que significa "limpieza de datos":
    Los datos del mundo real vienen con problemas: celdas vacias,
    numeros que no tienen sentido (una persona de 200 anos), texto
    que la computadora no puede procesar. La limpieza detecta y
    corrige esos problemas.

  Se hicieron 3 pasos:

  PASO 1 - Buscar celdas vacias (valores nulos):
    Se reviso columna por columna si habia celdas sin datos.
    Resultado: NO habia ninguna celda vacia. Cero nulos.

  PASO 2 - Eliminar datos que no tienen sentido (outliers):
    "Outlier" = un dato que se sale del rango logico.
    Se aplicaron estas reglas:
      - Si la edad es menor a 18 -> se elimina (un menor no puede
        pedir un credito)
      - Si la edad es mayor a 100 -> se elimina (probablemente es un
        error de carga)
      - Si el ingreso es negativo -> se elimina (es imposible)
      - Si la experiencia laboral es negativa -> se elimina (es imposible)

    Resultado: SE ELIMINARON 7 FILAS.
    Las 7 tenian edades fuera de rango (menores de 18 o mayores de 100).
    No se encontraron ingresos negativos ni experiencias negativas.
    El dataset quedo con 44,993 filas.

  PASO 3 - Convertir texto a numeros (codificacion):
    Los modelos de inteligencia artificial SOLO entienden numeros.
    No pueden leer "male" o "female". Entonces se convirtio:
      - female -> 0, male -> 1
      - High School -> 3, Bachelor -> 1, Master -> 4, etc.
      - No -> 0, Yes -> 1
    Se uso una herramienta llamada "LabelEncoder" de scikit-learn.

  Nota sobre la tasa de interes:
    La documentacion del dataset decia que la tasa de interes estaba
    en "puntos basicos" (ej: 1602 significaria 16.02%). Pero al
    revisar los datos reales, los valores ya venian en porcentaje
    (de 5.42% a 20.0%). Asi que NO se necesito hacer ninguna conversion.

SLIDE 9 - MODELOS DE CLASIFICACION (INTELIGENCIA ARTIFICIAL)
------------------------------------------------------------------------
Que poner en la slide:

  Que es un "modelo de clasificacion":
    Es un programa que aprende de datos del pasado para predecir
    el futuro. En este caso, ve miles de ejemplos de gente que
    pidio prestamos (algunos pagaron, otros no) y aprende a
    distinguir los patrones. Despues, cuando llega un solicitante
    nuevo, puede predecir si va a pagar o no.

  Como se dividieron los datos:
    No se puede evaluar un modelo con los mismos datos que uso
    para aprender (seria como hacer trampa en una prueba).
    Entonces se separaron los datos:
      - 70% para ENTRENAMIENTO (31,495 filas) -> el modelo aprende con estos
      - 30% para PRUEBA (13,498 filas) -> se evalua con estos que nunca vio

  Se probaron 2 modelos distintos:

    MODELO 1 - ARBOL DE DECISION (el ganador):
      Como funciona: es como un diagrama de flujo con preguntas.
      Ejemplo: "La tasa de interes es mayor a 13%?"
        -> Si: "El ingreso es menor a $30,000?"
          -> Si: PREDICE INCUMPLIMIENTO
          -> No: "Tiene incumplimientos previos?"
            -> Si: PREDICE INCUMPLIMIENTO
            -> No: PREDICE QUE VA A PAGAR
        -> No: PREDICE QUE VA A PAGAR

      Se limito a 5 niveles de preguntas para que no se complique
      demasiado (si le pones muchos niveles, memoriza los datos en
      vez de aprender patrones).

      Resultado: acerto el 91.28% de las veces.

    MODELO 2 - REGRESION LOGISTICA:
      Como funciona: usa una formula matematica que calcula un
      porcentaje de probabilidad. Por ejemplo, dice "este solicitante
      tiene 73% de probabilidad de incumplir". Si el porcentaje es
      mayor a 50%, predice que va a incumplir.

      Resultado: acerto el 88.54% de las veces.

    GANADOR: Arbol de Decision (91.28% vs 88.54%)
    Ademas, el arbol se puede VISUALIZAR (poner la imagen
    arbol_decision.png) lo que permite explicar POR QUE el modelo
    tomo cada decision. Esto se llama "interpretabilidad" y es
    un requisito del caso.

  Que significan los numeros (para las preguntas del profesor):

    "Accuracy" (precision general) = 91.28%
      De cada 100 predicciones, 91 son correctas.

    "Precision" = 0.88 para incumplidos
      De todos los que el modelo dijo "este va a incumplir",
      el 88% de verdad incumplio. Solo se equivoco en el 12%.

    "Recall" = 0.71 para incumplidos
      De todos los que REALMENTE incumplieron, el modelo logro
      detectar al 71%. Se le escapo el 29% restante.
      Por que no es mas alto? Porque el modelo prefiere no rechazar
      clientes buenos (ser preciso) antes que intentar detectar a
      todos los malos (y equivocarse mas).

SLIDE 10 - EQUIDAD (QUE EL MODELO NO DISCRIMINE)
------------------------------------------------------------------------
Que poner en la slide:

  Por que se analiza esto:
    Un modelo de IA NO puede funcionar mejor para hombres que para
    mujeres (o al reves). Si lo hace, estaria discriminando.
    La evaluacion pide especificamente que el modelo sea "justo".

  Que se hizo:
    Se midio la precision del modelo por separado para hombres y
    para mujeres, y se comparo.

  Resultados:
    - Hombres: el modelo acerto el 91.54% (con 7,506 casos de prueba)
    - Mujeres: el modelo acerto el 90.95% (con 5,992 casos de prueba)
    - Diferencia: solo 0.59 puntos porcentuales

  Criterio: si la diferencia es menor al 5%, se considera que
  el modelo es equitativo (justo).

  0.59% es MUCHO menor que 5%, asi que el modelo NO discrimina
  por genero. Funciona practicamente igual para ambos.

PREGUNTAS PROBABLES DEL PROFESOR (CON RESPUESTAS)
================================================================================

Leelas antes de la presentacion para estar preparado.

---

P: Por que se eligio el Arbol de Decision y no la Regresion Logistica?
R: Por dos razones. Primero, tiene mejor accuracy: 91.28% vs 88.54%.
   Segundo, es interpretable, se puede mostrar el arbol con las preguntas
   que hace el modelo y explicar exactamente por que aprueba o rechaza
   a alguien. La evaluacion pide que el modelo sea interpretable.

---

P: Por que se descartaron 7 filas? Como sabes que habia que eliminarlas?
R: Las 7 filas tenian edades menores a 18 o mayores a 100 anos. Un menor
   de edad no puede solicitar un credito legalmente, y una persona de
   mas de 100 anos es casi seguro un error de carga. Si los dejabamos,
   el modelo aprenderia patrones falsos. No habia nulos ni ingresos
   negativos.

---

P: Que pasa si llegan datos nuevos con celdas vacias?
R: El pipeline los detecta en la etapa de limpieza con la funcion
   df.isnull(). Las filas con valores nulos se eliminan automaticamente
   antes de que lleguen al modelo. Y queda registrado en el log cuantas
   se eliminaron.

---

P: Que es la validacion estructural vs la semantica?
R: Estructural revisa la FORMA de los datos: estan las columnas correctas?
   hay celdas vacias? cuantas filas hay?
   Semantica revisa el SIGNIFICADO: los numeros tienen sentido? la edad
   esta en un rango logico? el puntaje crediticio es valido? Es como
   revisar que un formulario este completo (estructural) vs revisar que
   lo que escribieron tenga sentido (semantica).

---

P: Es escalable esta solucion? Que pasa si hay millones de datos?
R: Si, es escalable. pandas puede manejar millones de filas. Si
   fueran cientos de millones, se podria migrar a herramientas mas
   potentes como Spark o Dask sin cambiar mucho el codigo. La base
   de datos SQLite se podria reemplazar por PostgreSQL o MySQL para
   produccion.

---

P: Que diferencia hay entre accuracy, precision y recall?
R: Accuracy es el porcentaje general de aciertos (91 de cada 100).
   Precision: cuando el modelo dice "este va a incumplir", que tan
   seguido tiene razon (88% de las veces).
   Recall: de toda la gente que de verdad incumplio, a cuantos logro
   detectar el modelo (71%). El 29% restante se le escapo.

---

P: Por que el recall es 71% y no mas alto?
R: Porque hay un equilibrio entre precision y recall. Si el modelo
   intentara detectar a TODOS los que incumplen, tambien rechazaria
   a muchos que SI iban a pagar (falsos positivos). Con 71% de recall
   y 88% de precision, el modelo tiene un buen equilibrio: detecta
   la mayoria de los riesgosos sin rechazar demasiados buenos clientes.

---

P: Que es DataOps? En que se diferencia de hacer las cosas normalmente?
R: DataOps es una metodologia que automatiza todo el trabajo con datos.
   Normalmente alguien descarga los datos, los limpia en Excel, hace
   un grafico y lo manda por mail. Con DataOps todo eso se hace con un
   solo comando, queda registrado, se puede repetir exactamente igual,
   tiene pruebas automaticas y monitoreo. Si cambian los datos, se
   reprocesa automaticamente.

---

P: Como garantizan que el modelo no discrimina?
R: Medimos la precision del modelo por separado para hombres y mujeres.
   La diferencia fue de solo 0.59%, muy por debajo del 5% que se usa
   como umbral. Eso significa que el modelo predice con la misma
   precision para ambos generos, no hay sesgo.
