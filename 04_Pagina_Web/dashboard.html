Que es este proyecto?

Una entidad financiera quiere predecir si un solicitante de credito va a incumplir el pago (no pagar el prestamo). Para eso, se tomo un dataset real de 45,000 solicitudes de prestamos con informacion como edad, ingresos, historial crediticio, monto del prestamo, etc.

El proyecto implementa un pipeline DataOps: un flujo automatizado que toma los datos crudos, los limpia, los valida, los almacena, los analiza visualmente y entrena modelos de inteligencia artificial para hacer la prediccion. Todo el proceso esta registrado con logs y metricas de monitoreo.

Etapa 1 — Ingesta de Datos

Que es la ingesta? Es el primer paso: leer los datos desde su fuente original y cargarlos en memoria para poder trabajar con ellos. En este caso, el archivo fuente es un CSV (Comma-Separated Values), un formato de texto plano donde cada fila es un registro y las columnas estan separadas por comas.

Se utilizo la libreria pandas de Python con la funcion pd.read_csv('datos/loan_data.csv'). Esta funcion:

  • Lee el archivo CSV linea por linea
  • Detecta automaticamente los tipos de datos (numeros, texto, etc.)
  • Lo convierte en un DataFrame (una tabla en memoria, como una hoja de Excel)

Resultado: se cargaron 45,000 filas con 14 columnas en 0.05 segundos.

# Codigo que hace la ingesta: df = pd.read_csv('datos/loan_data.csv') # df ahora tiene 45,000 filas x 14 columnas
Dato clave
El archivo original pesa aprox. 4 MB y contiene solicitudes de prestamos con datos como edad, ingreso, puntaje crediticio, monto solicitado y si la persona pago o no (loan_status).

Etapa 2 — Limpieza y Transformacion

Que es la limpieza? Los datos del mundo real vienen con problemas: valores faltantes, datos imposibles, texto que las maquinas no entienden. La limpieza es el proceso de detectar y corregir estos problemas para que los datos sean confiables antes de analizarlos.

Paso 1 — Buscar valores nulos (vacios):

Se reviso cada columna buscando celdas vacias. Resultado: 0 valores nulos. El dataset estaba completo en este aspecto.

Paso 2 — Eliminar outliers (valores atipicos):

Se aplicaron reglas de negocio para detectar datos que no tienen sentido logico:

  • person_age < 18 → Un menor de 18 no puede solicitar un credito
  • person_age > 100 → Edades irreales, posible error de carga
  • person_income < 0 → Un ingreso negativo no tiene sentido
  • person_emp_exp < 0 → Experiencia laboral negativa es imposible

Resultado: 7 filas eliminadas (todas por edades fuera de rango). El dataset quedo con 44,993 filas.

Paso 3 — Codificacion de variables categoricas:

Los modelos de machine learning solo entienden numeros. Las columnas con texto se convirtieron a numeros con LabelEncoder:

  • person_gender: female → 0, male → 1
  • person_education: Associate → 0, Bachelor → 1, Doctorate → 2, High School → 3, Master → 4
  • person_home_ownership: MORTGAGE → 0, OTHER → 1, OWN → 2, RENT → 3
  • loan_intent: DEBTCONSOLIDATION → 0, EDUCATION → 1, HOMEIMPROVEMENT → 2, MEDICAL → 3, PERSONAL → 4, VENTURE → 5
  • previous_loan_defaults_on_file: No → 0, Yes → 1
Por que se descartaron exactamente 7 filas?
De las 45,000 filas originales, 7 tenian edades menores a 18 o mayores a 100 anos. Estos son outliers (valores atipicos): datos que caen fuera del rango logico esperado. Incluirlos enganaria al modelo porque no representan solicitantes reales de credito. No se encontraron valores nulos ni ingresos/experiencias negativas.
Sobre la tasa de interes
El metadata del dataset indicaba que loan_int_rate estaba en puntos basicos (ej: 1602 = 16.02%), pero al revisar los datos reales los valores ya venian en porcentaje (rango: 5.42% - 20.0%, promedio: 11.01%). Por lo tanto, no se necesito conversion.

Etapa 3 — Validacion Estructural y Semantica

Que es la validacion? Despues de limpiar los datos, hay que verificar que todo quedo bien antes de usarlos. Es como un control de calidad en una fabrica: se revisa que el producto final cumple con las especificaciones. Se realizan dos tipos de validacion:

Validacion Estructural (la "forma" de los datos):

  • Estan las 14 columnas esperadas? → SI, todas presentes
  • Quedan valores nulos? → NO, 0 nulos
  • Cantidad de filas: 44,993
  • Cantidad de columnas: 14

Validacion Semantica (el "significado" de los datos):

  • Edades entre 18 y 100? → SI (min: 20, max: 94)
  • Ingresos no negativos? → SI (min: $8,000)
  • Credit score entre 300-850? → SI (min: 390, max: 784)
  • Tasa de interes entre 0-100%? → SI (min: 5.42%, max: 20.0%)
  • loan_status solo tiene 0 y 1? → SI, es binario
  • Completitud del dataset: 100%

Resultado: 0 errores de validacion. Los datos estan listos para ser usados.

Resultado
Todas las validaciones pasaron correctamente. La validacion estructural confirma que los datos tienen la forma esperada (columnas correctas, sin nulos). La validacion semantica confirma que los valores tienen sentido logico (edades reales, ingresos positivos, scores en rango valido).

Etapa 4 — Carga de Datos

Que es la carga? Una vez que los datos estan limpios y validados, se almacenan en formatos permanentes para que puedan ser consultados despues sin tener que repetir todo el proceso. Los datos se guardaron en dos formatos:

1. Archivo CSV limpio (../03_EDA_Resultados/datos_limpios.csv):

Una copia del dataset ya procesado en formato texto. Util para compartir o importar en otras herramientas como Excel o Google Sheets.

2. Base de datos SQLite (../03_EDA_Resultados/prestamos.db):

Una base de datos relacional ligera. Se creo la tabla prestamos con las 44,993 filas. Permite hacer consultas SQL sobre los datos, por ejemplo:

-- Cuantos prestamos de cada tipo hay? SELECT loan_intent, COUNT(*) FROM prestamos GROUP BY loan_intent; -- Promedio de ingreso de quienes incumplen? SELECT AVG(person_income) FROM prestamos WHERE loan_status = 1;

Se verifico que las 44,993 filas se cargaron correctamente en la base de datos.

Analisis Exploratorio de Datos (EDA)

Que es el EDA? Antes de entrenar un modelo de IA, hay que entender los datos visualmente. El EDA (Exploratory Data Analysis) genera graficos y estadisticas que permiten descubrir patrones, detectar anomalias y entender la distribucion de las variables. Es como "conocer" los datos antes de pedirle a la maquina que aprenda de ellos.

Modelos de Clasificacion

Que es un modelo de clasificacion? Es un algoritmo de inteligencia artificial que aprende de datos historicos para predecir a que categoria pertenece un nuevo dato. En este caso, predice si un prestamo sera pagado (0) o incumplido (1). Se entrenaron 2 modelos diferentes y se compararon para elegir el mejor.

Como se dividieron los datos?
Se separo el dataset en dos partes: 70% para entrenamiento (31,495 filas) y 30% para prueba (13,498 filas). El modelo aprende con el 70% y se evalua con el 30% que nunca vio, para medir que tan bien generaliza a datos nuevos.

Que significan estas metricas?
Accuracy: Porcentaje de predicciones correctas sobre el total. Precision: De todos los que el modelo dijo que incumplirian, cuantos realmente incumplieron. Recall: De todos los que realmente incumplieron, cuantos detecto el modelo. Un recall de 0.71 significa que el modelo detecta el 71% de los incumplimientos reales.

Como replicar este proyecto

Sigue estos pasos para ejecutar el pipeline completo en tu computadora y obtener los mismos resultados.

Requisitos previos:

  • Python 3.10 o superior instalado
  • pip (gestor de paquetes de Python)
  • Git (para clonar el repositorio)

Paso 1 — Clonar el repositorio:

git clone <url-del-repositorio> cd GestionDeDatos

Paso 2 — Crear un entorno virtual (recomendado):

# Crea un entorno aislado para no afectar otras instalaciones de Python python -m venv venv # Activar en Windows: venv\Scripts\activate # Activar en Linux/Mac: source venv/bin/activate

Paso 3 — Instalar dependencias:

pip install -r 02_Ingesta_y_Pipeline/requirements.txt # Instala: pandas, scikit-learn, matplotlib, seaborn, pytest

Paso 4 — Ejecutar el pipeline completo:

python 02_Ingesta_y_Pipeline/analisis.py # Ejecuta las 4 etapas + EDA + modelos + equidad # Los resultados se guardan en la carpeta 03_EDA_Resultados/

Paso 5 — Ejecutar los tests automatizados:

pytest 02_Ingesta_y_Pipeline/test_analisis.py -v # Ejecuta 5 tests que verifican: # - Que el archivo de datos existe # - Que los datos se cargan correctamente # - Que las 14 columnas esperadas estan presentes # - Que los rangos de datos son validos # - Que la variable objetivo es binaria (0/1)

Paso 6 — Ver resultados:

Abre 04_Pagina_Web/dashboard.html en un navegador para ver este dashboard, o revisa la carpeta 03_EDA_Resultados/:
